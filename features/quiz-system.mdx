## Overview

The quiz system allows users to test their Scripture knowledge through interactive multiple-choice quizzes generated from scrolls or custom passages.

<Tabs>

## Quiz Taking Interface

### Progress Tracking

- Question counter (e.g., "Question 3 of 5")
- Progress bar showing completion
- Previous/Next navigation buttons
- Navigation disabled until answer selected

### Question Interface

<Accordion title="Custom Options Design">
  - Button-based options (not radio buttons)
  - **Letter prefixes**: A, B, C, D in circular badges
  - Selection states with distinct visual feedback
  - Background, border, and text color changes
  - Hover scale animations
  - Smooth transitions
</Accordion>

<Accordion title="Question Details">
  - Question text with verse references
  - Multiple choice options (typically 4)
  - Letter indicators for each option
  - Clear selection state
</Accordion>

## Quiz Results

After completion, users see comprehensive results:

<Steps>
  <Step title="Score Display">
    Large percentage score prominently displayed
  </Step>

  <Step title="Review Your Answers">
    Per-question review section with:
    - ✅ **Correct answers**: Green background, green border, checkmark
    - ❌ **Incorrect answers**: Red background, red border, X mark
    - Explanation boxes with verse references
    - Your answer vs correct answer highlighted
  </Step>

  <Step title="Next Steps">
    - "Retake Quiz" button to try again
    - Return to Scripture Explorer
    - View related scroll (if from scroll)
  </Step>
</Steps>

### Uniform Results Style

<Tip>
  Quiz results use consistent styling across the application (Daily Quiz and Scroll quizzes)
</Tip>

## Integration with Scrolls

Quizzes are tightly integrated with the Scroll system:

```
Scroll Detail View
  ↓
  Quiz Status: Shows previous score if taken
  ↓
  "Take Quiz" button → /scroll/:id?startQuiz=true
  ↓
  Quiz interface within scroll context
  ↓
  Results saved to scroll metadata
```

<Accordion title="Quiz Status Badges">
  Scrolls display quiz information:
  - "Quiz Available" - Not yet attempted
  - "Score: 80%" - Previous attempt with percentage
  - Color-coded badges (green for passed, yellow for needs improvement)
</Accordion>

## Data Structure

```typescript
interface QuizQuestion {
  id: string;
  question: string;
  options: string[];
  correctAnswer: number;  // Zero-based index
  explanation?: string;
  verseReference?: string;
}

interface Quiz {
  id: string;
  scrollId: string;
  title: string;
  questions: QuizQuestion[];
}
```
